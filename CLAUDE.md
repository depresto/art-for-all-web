## Commands

```bash
pnpm dev              # Start dev server (localhost:3000)
pnpm build            # Production build
pnpm start            # Start production server
pnpm lint             # ESLint (next/core-web-vitals)
npx prisma generate   # Regenerate Prisma client after schema changes
npx prisma db pull    # Introspect DB and update schema.prisma
npx prisma studio     # Open Prisma Studio (DB browser)
```

Pre-commit hooks (via Husky + lint-staged) automatically run on `*.{ts,tsx}` files: TypeScript check, ESLint fix, organize-imports-cli, Prettier.

## Architecture

Full-stack Next.js 14 (Pages Router) application for an art platform. Package manager is **pnpm**.

### Frontend

- **Routing**: File-based in `src/pages/`. Dynamic routes include `artwork/[artworkId].tsx`.
- **UI**: Ant Design 5 components + styled-components for customization. Shared styled components live in `src/components/common/index.tsx`.
- **State**: React Context API — `AuthContext` (JWT auth state), `ApiContext` (SWR config), `AudioPlayerContext` (audio player).
- **Provider nesting** (`src/Application.tsx`): AppContainer → AuthProvider → ApiProvider → AudioPlayerProvider → Children.
- **Data fetching**: SWR via `/api/v2/` REST endpoints. Custom hooks in `src/hooks/` (useArtwork, useCollection, useExhibition, useAudioBook, useTag) wrap SWR calls. Hook signatures remain stable — pages do not need to change when hooks are updated.
- **SVGs**: Imported as React components via @svgr/webpack.

### Backend (API Routes)

- **Auth API**: `src/pages/api/v1/[[...slug]].ts` — catch-all route using next-connect for auth endpoints.
- **Data API**: `src/pages/api/v2/` — REST endpoints for artworks, exhibitions, collections, comments, audio-books, tags.
- **Middleware chain** (v1): CORS → express-session (Redis-backed) → JWT validation → Router.
- **Auth flow**: JWT tokens (signed with `JWT_SECRET`) + Redis session storage. Tokens include memberId, role, email. Auto-refresh via `/api/v1/auth/refresh-token`.
- **Server code**: `src/server/` contains `router/`, `controllers/`, `middlewares/`, `helpers/`.

### Data Layer

- **Prisma ORM** connects directly to PostgreSQL via `DATABASE_URL`.
- **Schema**: `prisma/schema.prisma` — uses `@@map()` to map PascalCase models to snake_case table names.
- **Client singleton**: `src/lib/prisma.ts` — includes BigInt JSON serialization and dev hot-reload protection.
- **API utilities**: `src/lib/api-utils.ts` — JWT extraction helpers (`getMemberIdFromRequest`, `requireMemberId`).
- **Generated client**: `src/generated/prisma/` — auto-generated by `npx prisma generate`. Do not edit manually.

### SSR Styling

`_document.tsx` integrates both @ant-design/cssinjs and styled-components style collection for server-side rendering.

## Code Style

- Prettier: single quotes, no semicolons, trailing commas, 120 char width, no arrow parens
- TypeScript strict mode
- Styling: prefer styled-components wrapping Ant Design components
